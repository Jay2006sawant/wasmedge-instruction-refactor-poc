## M1.1 Instruction + Immediate Mapping (Loader -> AST -> Validator -> Executor)

### Purpose
This document records my analysis of how WasmEdge actually handles instructions + immediates from binary parsing Ôćĺ loader Ôćĺ AST Ôćĺ validator Ôćĺ executor Ôćĺ tests. This is the groundwork for Milestone 1's opcode/immediate split refactor.

### 1) What `AST::Instruction` actually stores (immediate storage model)
WasmEdgeÔÇÖs interpreter instruction node is `WasmEdge::AST::Instruction`. It contains:

1. `OpCode Code` (the decoded opcode).
2. `uint32_t Offset` (byte offset of the instruction in the module; used for diagnostics).
3. A tagged union `Data` whose active arm depends on opcode category.
4. A small `Flags` struct that stores allocator booleans + `MemLane` for SIMD memory lane ops.

The union arms map directly to ÔÇťimmediate familiesÔÇŁ:
* Block control immediates: `Data.Blocks = { JumpEnd, JumpElse, ResType }`
* Branch/jump immediates: `Data.Jump` (`JumpDescriptor`)
* Br_table label list: `Data.BrTable = { LabelListSize, LabelList* }`
* Select_t result type list: `Data.SelectT = { ValTypeListSize, ValType* }`
* Try_table payload: `Data.TryCatch` (`TryDescriptor*`)
* Local/global indices + stack offsets: `Data.Indices = { TargetIdx, SourceIdx, StackOffset }`
* Memory args: `Data.Memories = { TargetIdx, MemAlign, MemOffset }`
* Numeric constants: `Data.Num`
* End flags: `Data.EndFlags = { IsExprLast, IsTryBlockLast }`
* Branch-on-cast payload: `Data.BrCast` (`BrCastDescriptor*`, includes `JumpDescriptor Jump + RType1/RType2`)

Additionally, SIMD memory lane ops store `MemLane` in `Flags.MemLane` (outside the union) to keep the ÔÇťmemory instruction hot pathÔÇŁ denser.

### 2) Loader behavior: what it parses and where it writes
The loader reads opcode-by-opcode and calls `loadInstruction(Instr)` to fill the instructionÔÇÖs immediate storage.

#### Control-flow immediates
* `block`, `loop`, `if`, `try_table`
  * Loader fills `Data.Blocks.ResType` via `readBlockType(...)`.
* `else`
  * Loader does not parse new immediates; instead it retroactively writes the computed jump distance into the matching `if` instruction: `Instrs[Pos].setJumpElse(...)`.
* `end`
  * Loader retroactively writes jump distances and ÔÇťend-of-expressionÔÇŁ bookkeeping:
    * `Instrs.back().setExprLast(...)` and `setTryBlockLast(...)`
    * sets `JumpEnd` on the corresponding block/loop/if or `Try_table` container.
* `br`, `br_if`, `br_on_null`, `br_on_non_null`
  * Loader writes `Instr.getJump().TargetIndex` from a single `u32` label index.
* `br_table`
  * Loader writes a variable-length label list into `Instr.getLabelList()`:
    * reads `VecCnt` then `VecCnt` label indices
    * reads default label index
    * allocates `LabelListSize = VecCnt + 1` and fills all entries.
* `br_on_cast`, `br_on_cast_fail`
  * Loader reads a `u8` flag byte, a `u32` label index, and two reference heap type immediates.
  * It sets `Instr.setBrCast(LabelIdx)` and then writes `BrCastDescriptor.RType1` and `RType2`.
* `try_table`
  * Loader sets `Instr.setTryCatch()`, parses the result type, then parses each catch clause:
    * reads catch flags (`IsRef`, `IsAll`)
    * reads `TagIndex` only when `!IsAll`
    * reads `LabelIndex` for every catch clause

#### Index-based immediates
* `local.get`, `local.set`, `local.tee`, `global.get`, `global.set`
  * Loader writes `Instr.getTargetIndex()` from `u32`.
* `call`, `return_call`, `call_ref`, `return_call_ref`
  * Loader writes `Instr.getTargetIndex()` from `u32`.
* `call_indirect`, `return_call_indirect`
  * Loader writes `Instr.getTargetIndex()` (type index) and `Instr.getSourceIndex()` (table index).
* Table instructions (`table.get/set/init/copy/...`)
  * Loader writes `Instr.getTargetIndex()` and/or `Instr.getSourceIndex()` depending on instruction form.
* `elem.drop`, `memory.init`, `data.drop`, etc.
  * Loader writes the relevant index fields (`TargetIndex`/`SourceIndex`) using `readU32(...)`.

#### Type-vector / heap-type immediates
* `select_t`
  * Loader reads `VecCnt` and then loads each `ValType`, storing them in `Instr.getValTypeList()`.
* Reference immediates
  * Many reference instructions store heap types into `Instr.getValType()`.
  * `br_on_cast*` stores heap types into `Instr.getBrCast().RType1/RType2`.

#### Memory immediates
* Memory load/store forms (including SIMD memory loads/stores)
  * Loader fills:
    * `Instr.getMemoryAlign()` (via `readMemImmediate()`; align may be adjusted based on proposal flags)
    * `Instr.getMemoryOffset()` (u32 or u64 depending on proposals)
    * `Instr.getTargetIndex()` (memory index; for multi-memories this may be embedded in the align flags)
  * For lane-specific SIMD memory ops:
    * Loader additionally writes `Instr.getMemoryLane()` from a `u8` lane index.

#### Numeric constants
* `i32.const`, `i64.const`, `f32.const`, `f64.const`
  * Loader stores the decoded immediate into `Instr.setNum(...)` (via `ValVariant` conversion).

### 3) Validator behavior: what it reads and what it mutates
In WasmEdge, validation is not only ÔÇťread-onlyÔÇŁ: it derives instruction-local metadata and writes it back into the `AST::Instruction` immediates.

#### Control-flow validation mutations
The `FormChecker` validates instructions using a stack of control frames. For several branch-like opcodes it records jump metadata into the `JumpDescriptor` stored in the instruction.
* Key mutation helper: `recordJump(Jump, Arity, D)`
  * Writes:
    * `Jump.StackEraseBegin`
    * `Jump.StackEraseEnd`
    * `Jump.PCOffset`
  * Applies to:
    * `br`, `br_if`
    * `br_table` entries
    * `br_on_null`, `br_on_non_null`
    * `br_on_cast*` (using `Instr.getBrCast().Jump`)
    * catch handlers in `try_table` (writes into `CatchDescriptor.Jump`)

#### Local variable validation mutations
* `local.get`, `local.set`, `local.tee`
  * Validator computes and writes `Instr.getStackOffset()` using the current `ValStack` and the locals vector.
  * It also updates initialization state (e.g., `LocalInits`).

#### Try_table validation mutations
* `try_table` validation sets derived handler metadata:
  * `TryDescriptor.BlockParamNum` is derived from the computed handler stack type/arity.

#### Memory validation (checks)
* For memory instructions, validator checks:
  * alignment is within constraints and is consistent for atomic vs non-atomic families
  * offset bounds (`Instr.getMemoryOffset()`) are within the memoryÔÇÖs max address
  * lane index correctness when applicable (for SIMD lane checks)
* The memory immediates (`MemAlign`, `MemOffset`, `MemLane`) are treated as inputs for checking during validation; validator does not rewrite them as part of this phase.

### 4) Executor behavior: what it reads at run-time
At interpreter dispatch time, `engine.cpp` calls specialized handlers that read immediates from `AST::Instruction`.

#### Control-flow reads
* `if/else/end` handler:
  * reads `Instr.getJumpElse()` and `Instr.getJumpEnd()` to move the program counter.
* `br*` handler:
  * reads `Instr.getJump()` and passes it into `branchToLabel(...)`.
  * `branchToLabel(...)` reads:
    * `JumpDesc.StackEraseBegin` / `JumpDesc.StackEraseEnd` to erase the operand stack slice
    * `JumpDesc.PCOffset` to adjust PC
* `br_table` handler:
  * reads `Instr.getLabelList()` and selects one of the label entries based on the runtime stack value.
* `br_on_cast*` handler:
  * reads `Instr.getBrCast().RType2` and `Instr.getBrCast().Jump`.
* `try_table` handler:
  * reads `TryDescriptor.BlockParamNum` and `TryDescriptor.Catch` to push an exception handler.

#### Variable / index reads
* `local.get/set/tee` handlers:
  * read `Instr.getStackOffset()` and use it to address locals in the runtime stack frame.
* `global.*`, `call*`, table operations:
  * read `TargetIndex` / `SourceIndex` fields (`Instr.getTargetIndex()`, `Instr.getSourceIndex()`) to locate runtime instances and types.

#### Memory reads
* Scalar `load` / `store` helpers:
  * `runLoadOp` reads `Instr.getMemoryOffset()` when computing the effective address (`EA = addr + offset`).
  * `runStoreOp` reads `Instr.getMemoryOffset()` when computing `EA = addr + offset`.
  * executorÔÇÖs memory helpers rely on offsets computed/validated earlier.
* SIMD lane memory helpers:
  * `runLoadLaneOp` reads:
    * `Instr.getMemoryLane()` (lane selection, endian-adjusted)
    * `Instr.getMemoryOffset()` (effective address)
  * `runStoreLaneOp` reads:
    * `Instr.getMemoryLane()` (lane selection, endian-adjusted)
    * `Instr.getMemoryOffset()` (effective address)

#### Numeric constant reads
* `*const` handlers push the immediate via `Instr.getNum()` into the runtime stack.

### 5) Immediate-family read/write matrix (summary for M1.1)
The table below summarizes who writes and who reads for each immediate family.

| Immediate family (in `AST::Instruction`) | Loader writes | Validator reads/writes | Executor reads | Evidence/tests |
|---|---|---|---|---|
| `Offset` (diagnostic byte offset) | set `Instruction(Code, Off)` constructor | reads for errors | reads only for diagnostics | loader tests (indirect), error logs |
| BlockType / ResType (`Data.Blocks.ResType`, `TryDescriptor.ResType`) | parses block/try result type | reads for type-checking | reads indirectly via handler opcode semantics | loader instruction tests |
| `JumpDescriptor` (`Data.Jump`, `CatchDescriptor.Jump`, `BrCastDescriptor.Jump`) | sets `TargetIndex` for br-like opcodes | mutates `StackEraseBegin`, `StackEraseEnd`, `PCOffset` via `recordJump` | reads `StackErase*` and `PCOffset` via `branchToLabel` | validator + executor parity via spec tests |
| `BrTable` label list (`Data.BrTable.LabelList`) | allocates + fills label indices | reads label entries and mutates per-entry `Jump*` via `recordJump` | reads label list and branches by runtime index | loader tests + serialize tests |
| `Select_t` type list (`Data.SelectT.ValTypeList`) | parses vector of `ValType` | reads list to validate stack types | reads only through stack/type semantics | loader tests + validator gates |
| Indices (`TargetIdx`, `SourceIdx`) | parses label/table/memory/function/global indices | reads for bounds/type checks | reads to locate instances/types | loader tests + spec tests |
| Local stack offset (`Data.Indices.StackOffset`) | none at parse time | mutates `StackOffset` for local get/set/tee | reads `StackOffset` to load/store locals | validator + executor parity |
| Memory args (`MemAlign`, `MemOffset`, `TargetIdx`, `MemLane`) | parses align/offset/index and lane index for lane ops | reads and checks alignment/offset/lane constraints | reads `MemOffset` (all loads/stores) and `MemLane` (lane ops) | loader + serializer tests; executor memory helper paths |
| Numeric constants (`Data.Num`) | parses numeric immediates into `Num` | reads indirectly via stack type semantics | reads via `Instr.getNum()` push | loader + serializer tests |
| `TryDescriptor` / catch payload | parses catch flags, tag/label indices, result type | mutates `BlockParamNum` and records handler jump metadata | reads `Catch` and uses handler to branch/execute | loader instruction tests + executor EH tests |
